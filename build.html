What inside?

Basic commands

gulp
this will compile the entire project into development mode, start browserSync on localhost and will watch all the changes to the SCSS, html, images etc. 
gulp dist
this is used to compile project assets into production. 
gulp zip
compress all files into zip (except node_modules folder). The name of ZIP is the same as the project folder.

Additional commands for CSS compilation:

gulp compact
gulp compressed
gulp nested

NOTE: this is for css output control

Project structure

gulp folder with gulp tasks
src folder with source files
babel.config.js configuration file for Babel (es6 code transpiler)
eslintrc.js configuration file Eslint
gulpfile.js GULP main file
htmlhint.config.js configuration file for HTML hint
package.json project dependencies
prettier.config.js configuration file for Prettier (code formatter)
README.md Short guide for build
sass-lint.yml configuration file for SASS/SCSS linter
webpack.config.js configuration file for webpack (if chosen)
P.S. all files are mandatory. everything should be included into the zip for the client.

webpack is a static module bundler for modern JavaScript applications. When webpack processes your application, it internally builds a dependency graph which maps every module your project needs and generates one or more bundles.

What is a webpack Module?

An ES2015 import statement

import "./module-name";
import "module-name";
import defaultExport from "module-name";
import jQuery from "jquery";
import foo from "../../bar";
ES2015 import In contrast to Node.js modules, webpack modules can express their dependencies in a variety of ways. A few examples are: module-name The module to import from. This is an example of absolute path name to the .js file containing the module. module-name This is an example of relative path name to the .js file containing the module. By contrast, a relative path starts from some given working directory, avoiding the need to provide the full absolute path (in this case - node_modules folder) defaultExport Name that will refer to the default export from the module. Real example of default export In this example, we are assigning a name for jQuery module. If you will use jQuery throught your module code, assing a name, otherwise - you can use 'unnamed' import example above.
ES2015 import usage example
bar.js 
export default function bar() {
  console.log('Module `bar` exported');
}
app.js 
import foo from './bar';

foo();
console.log('Module `bar` imported');

A CommonJS require() statement
const package = require('module-name')
const { a, b, c } = require('./bar')
The CommonJS module specification is the standard used in Node.js for working with modules. Client-side JavaScript that runs in the browser uses another standard, called ES Modules Modules are very cool, because they let you encapsulate all sorts of functionality, and expose this functionality to other JavaScript files, as libraries. They let you create clearly separate and reusable snippets of functionality, each testable on its own. The huge npm ecosystem is built upon this CommonJS format. The syntax to import a module is:

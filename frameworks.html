HOW TO: Frameworks

For what they are developed? what problem do they solve

Frameworks for the rescue!

Creating consistent, and effective CSS can be quite a lot of work. There are so many things to consider like responsiveness, accessibility, and structure. This is exactly why CSS Frameworks exist, to take that routine off of you!

They are pre-tested, crossbrowser, documented and worldwide usable.

To summarize: there’s no need to reinvent the wheel.

How to understand:

what to change?
where to find?
what is that weird thing and how to change it?

  1. Read the documentation
  2. Inspect the source files of the framework
  3. Understand the architecture of framework
  4. Inspect examples
  5. Find source files for the components
  6. Check all the available variables in the framework (yes, scroll the whole file and read)
  7. Check component based variables. What you can to change? What result is expected?
  8. Find, change, check - repeat!

Things to do:

Check all the designs

  • Open each page, find colors/font-sizes/line-heights/etc
  • Check repeating components
  • Make a plan of development process, order of customization/creation/modification of components

Define global variables

  • Typography
  • Colors
  • Spacings
  • General components
  • Repeating components etc.

Make sure that you know which component should be "used here"

  • Open documentation side-by-side to the design
  • Try to find closest component to the design and check how you can modify it
  • Think about the modification of the component

Follow framework architecture in terms of custom component creation.

  • Custom variables - not for framework components only!
  • While creating new component - keep the same code style, class naming-convention, code structure
  • Add custom variables for your component into the file with custom framework variables.
  • Keep well structured blocks of variables, group them by logic and type of CSS property

Closer to coding.

What is the main Bootstrap css unit?

There is very handy function in our functions file called rem-calc();

Function itself


// @location: base/_functions.scss
// there is also em-calc function

@function rem-calc($values, $base-value: $rem-base) {
  $max: length($values);

  @if $max == 1 {
    @return convert-to-rem(nth($values, 1), $base-value);
  }

  $rem-values: ();
  @for $i from 1 through $max {
    $rem-values: append($rem-values, 
                        convert-to-rem(nth($values, $i), 
                        $base-value));
  }
  @return $rem-values;
}

How to use


// $rem-base: 10px;

.card__thumbnail {
  min-height: rem-calc(137);
  position: relative;
  overflow: hidden;
}

Result in CSS


.card__thumbnail {
  min-height: 13.7rem;
  position: relative;
  overflow: hidden;
}

Bootstrap theming: conclusion

  1. Only colors that you need should be used as a theme. Remember, that CSS output much longer then SCSS
  2. Always use SCSS power: variables, maps, loops, mixins, nesting, extends, everything - that will help you to create expandable, editable, understandable code.
  3. Always think about feedback: request to change color, request to remove component from theming, request to add colors, etc.
  4. Better to keep this in separate file: easier to add/remove
  5. Even if there is no comments in requirements - try to document HOW TO USE this. This is will be a huge Plus to your carma.
  6. Try to found best "USE CASE" in this particular project. One approach can be perfect for this project - but bad for another.
  7. Research for different approaches, blogs, etc. Always improve yourself as a developer. Dont be "straightforward" developer.
